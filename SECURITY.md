# 安全策略

本文说明如何报告 Bennett Toolbox 的安全问题，以及哪些信息有助于定位和
修复问题。

## 支持范围

安全修复优先面向 `master` 分支的最新代码。安装包由 GitHub Actions 从
`master` 分支自动构建。

## 报告安全问题

如果你发现安全问题，请不要创建公开 GitHub Issue。请使用以下方式私下报告：

- 邮箱：bennettmail@163.com
- GitHub 私有漏洞报告功能，如果仓库已开启该能力。

报告时请尽量提供以下信息：

- 受影响的操作系统。
- 受影响的应用版本或提交 SHA。
- 复现步骤。
- 问题影响。
- 复现所需的输入文件或本地环境说明。

## 问题范围

项目会处理用户主动选择的本地文件。以下类型的问题属于重点关注范围：

- 文件解析导致的异常行为。
- 打包依赖中的不安全行为。
- 本地数据意外暴露。
- 非预期的网络访问。
- Electron 主进程和渲染进程之间的数据边界问题。

如果问题依赖应用之外的任意本地代码执行，通常不属于本项目安全范围，除非
它展示了 Bennett Toolbox 引入的新风险。
